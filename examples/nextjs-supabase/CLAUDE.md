## Project Overview

Launchpad is a B2B SaaS platform that helps early-stage startups manage investor relationships, track fundraising pipelines, and generate board reports. Primary users are founders and their executive assistants.

**Stack:**
- **Framework:** Next.js 14 (App Router)
- **Language:** TypeScript
- **Database:** Supabase (PostgreSQL + Auth + Storage)
- **UI Library:** shadcn/ui + Tailwind CSS
- **Deployment:** Vercel
- **Testing:** Playwright E2E
- **AI:** OpenAI GPT-4o (report generation)

## Next.js App Router Conventions

- Pages live in `app/` using the App Router
- Server components by default -- add `'use client'` only when needed
- API routes in `app/api/` as Route Handlers
- Shared types in `types/` directory
- Server Actions in `app/actions/` for form submissions
- Environment variables: server-side in `.env.local`, client-side must be prefixed `NEXT_PUBLIC_`

## Supabase Conventions

- Every migration generated with `supabase db diff -f <migration_name>`
- RLS enabled on every user-facing table -- default deny
- Type generation: `supabase gen types typescript --local > types/database.ts`
- Auth: use `@supabase/ssr` for server-side session handling
- Storage buckets: `avatars` (public), `documents` (private, RLS required)

## shadcn/ui Conventions

- Component additions: `npx shadcn@latest add <component>`
- Custom variants go in the component file itself, not globals
- Color tokens defined in `tailwind.config.ts` -- do not use raw hex in components
- Dark mode: class-based via `next-themes`
