## Project Overview

Launchpad is a B2B SaaS platform that helps early-stage startups manage investor relationships, track fundraising pipelines, and generate board reports. Primary users are founders and their executive assistants.

- **UI Library:** shadcn/ui + Tailwind CSS

## Verification Gate

**Iron Law: NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE**

Before claiming any task is done, fixed, or passing:
1. IDENTIFY what command proves the claim
2. RUN it (fresh and complete)
3. READ the full output and check exit code
4. VERIFY output actually confirms the claim
5. ONLY THEN make the claim, and include the output

| Claim | Required evidence |
|-------|-----------------|
| Tests pass | `npm test` output: 0 failures |
| Linter clean | `npm run lint` output: 0 errors |
| E2E passing | `npx playwright test` output: all passing |
| DB migration applied | `supabase db push` ran, query confirms schema |
| Deployment live | Vercel URL + CI status green |

Banned phrases without fresh evidence: "should work", "try it now", "looks correct", "should be fixed".

## Next.js App Router Conventions

- Pages in `app/` using App Router
- Server components by default — add `'use client'` only when needed
- API routes in `app/api/` as Route Handlers
- Server Actions in `app/actions/` for form submissions
- Shared types in `types/`
- Environment: server-side in `.env.local`, client-side must be `NEXT_PUBLIC_` prefixed

## Supabase Conventions

- Migrations via `supabase db diff -f <migration_name>`
- RLS on every user-facing table — default deny
- Type generation: `supabase gen types typescript --local > types/database.ts`
- Auth: `@supabase/ssr` for server-side session handling

## shadcn/ui Conventions

- Add components: `npx shadcn@latest add <component>`
- Color tokens in `tailwind.config.ts` — no raw hex in components
- Dark mode: class-based via `next-themes`
