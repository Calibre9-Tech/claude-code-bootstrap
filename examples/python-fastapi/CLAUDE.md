## Project Overview

Hermes is a document processing API that ingests PDFs, extracts structured data using LLMs, and exposes the results via a REST API. Used internally by operations teams to automate data entry from vendor invoices and contracts.

**Stack:**
- **Framework:** FastAPI 0.111
- **Language:** Python 3.12
- **Database:** PostgreSQL 16 (via SQLAlchemy 2 + Alembic)
- **Queue:** Redis + Celery (async document processing)
- **Deployment:** Docker + AWS ECS
- **Testing:** pytest + httpx (unit + integration)
- **AI:** OpenAI GPT-4o (structured extraction)

## FastAPI Conventions

- Route files in `src/api/routes/` -- one file per resource (`documents.py`, `jobs.py`)
- Pydantic models in `src/schemas/` -- separate request and response models
- Business logic in `src/services/` -- routes call services, services call repositories
- Database access in `src/repositories/` -- no raw SQL in routes or services
- Dependency injection via FastAPI `Depends()` for auth and DB sessions
- All endpoints require authentication except `/health` and `/docs`

## SQLAlchemy / Alembic Conventions

- Models in `src/models/` -- one file per table group
- Always use `uuid` primary keys: `id: Mapped[UUID] = mapped_column(default=uuid4, primary_key=True)`
- Timestamps: `created_at` and `updated_at` with server defaults
- Migration command: `alembic revision --autogenerate -m "<description>"`
- Apply: `alembic upgrade head`
- Never edit a migration file after it has been applied to any environment

## Testing Conventions

- Unit tests in `tests/unit/` -- mock all I/O
- Integration tests in `tests/integration/` -- use test database, real HTTP calls via `httpx.AsyncClient`
- Fixtures in `tests/conftest.py`
- Test database is spun up via `docker-compose.test.yml`
- Run all: `pytest` | Run unit only: `pytest tests/unit/`
