## Project Overview

Campfire is a community discussion platform for small hobby groups. Members can post updates, share photos, and coordinate meetups. Think Discord-lite but simpler -- no bots, no servers, just groups of friends. Users are non-technical adults aged 35-65.

**Stack:**
- **Framework:** React 18 (Vite)
- **Language:** TypeScript
- **Database:** Firebase Firestore + Firebase Storage
- **Auth:** Firebase Authentication (email/password + Google)
- **UI Library:** Tailwind CSS + Headless UI
- **Deployment:** Netlify
- **Testing:** Vitest (unit) + Playwright (E2E)

## React Conventions

- Components in `src/components/` -- feature folders under `src/features/`
- Custom hooks in `src/hooks/` -- prefixed with `use`
- Context providers in `src/context/`
- Firebase SDK initialized once in `src/lib/firebase.ts`
- No direct Firestore calls in components -- use hooks or services layer
- All forms use React Hook Form

## Firestore Data Model

```
groups/{groupId}
  posts/{postId}
    comments/{commentId}
  members/{userId}

users/{userId}
  notifications/{notificationId}
```

Rules:
- Members can read group content
- Only the author can delete their own posts/comments
- Group admin can remove any content

## Firebase Emulator

Always test with the emulator before deploying rules:

```bash
firebase emulators:start          # Start all emulators
firebase emulators:start --only firestore,auth   # Just Firestore + Auth
```

Emulator UI: http://localhost:4000

## Tailwind / Headless UI Conventions

- Colors: `brand-*` tokens (defined in `tailwind.config.ts`) -- no raw hex
- Spacing: use Tailwind scale -- avoid custom values unless necessary
- Mobile-first: write base styles for mobile, add `md:` and `lg:` variants
- All dialogs and dropdowns use Headless UI primitives
